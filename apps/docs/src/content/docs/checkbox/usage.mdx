## Usage

Import the component:

```tsx
import { Checkbox, CheckboxLabel, CheckboxRootProvider } from "~/components/checkbox";
```

Basic checkbox:

```tsx
<Checkbox />
```

With label:

```tsx
<Checkbox defaultChecked>
  <CheckboxLabel>Accept terms</CheckboxLabel>
</Checkbox>
```

Disabled:

```tsx
<Checkbox disabled />
```

Indeterminate:

```tsx
<Checkbox checked="indeterminate" />
```

### Root Provider

Use <InlineCode>CheckboxRootProvider</InlineCode> when you need to access the checkbox state outside of the checkbox tree. This pattern uses the <InlineCode>useCheckbox</InlineCode> hook from Ark UI to create a shared context that both the checkbox and external elements can reference.

<CheckboxRootProviderDemo />

```tsx
import { useCheckbox } from "@ark-ui/solid/checkbox";
import { CheckboxRootProvider, CheckboxLabel } from "~/components/checkbox";

export function CheckboxWithExternalControl() {
  const checkbox = useCheckbox({ defaultChecked: true });

  return (
    <div>
      {/* Access checkbox state outside the tree */}
      <output>Checked: {JSON.stringify(checkbox().checked)}</output>

      <CheckboxRootProvider value={checkbox}>
        <CheckboxLabel>Subscribe to newsletter</CheckboxLabel>
      </CheckboxRootProvider>
    </div>
  );
}
```

The key difference:

- **<InlineCode>Checkbox</InlineCode> (Root)** — manages its own state internally. Use for simple, self-contained checkboxes.
- **<InlineCode>CheckboxRootProvider</InlineCode>** — accepts a pre-created checkbox context via <InlineCode>useCheckbox</InlineCode>. Use when you need to read or control the checkbox state from outside the component tree.
